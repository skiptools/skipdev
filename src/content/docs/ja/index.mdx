# Skip の仕組み

Swift および SwiftUI ソースを作成すると, Skip Xcode プラグインは Android 用の同等のコードを継続的にビルドします。両方のプラットフォームでネイティブ パフォーマンスとネイティブ ユーザー インターフェイスを備えたデュアルプラットフォーム ライブラリまたはアプリ全体を開発します。


## Skip の利点

<div class="feature-grid">

  <FeatureCard title="SwiftUI、隅々まで" icon="widgets" color="blue" link="/docs/modules/skip-ui/#supported-swiftui">
    本物の SwiftUI を一度記述すれば、両方のプラットフォームでファーストクラスのネイティブ UI を入手できます。
    Skip は SwiftUI ビューを Jetpack Compose に変換し、レイアウト、
    アクセシビリティ、プラットフォームの規則を維持します。
  </FeatureCard>

  <FeatureCard title="ランタイムなし。肥大化なし。" icon="bolt" color="amber" link="/docs/modes/">
    Skip はコンパイル時にその作業を行います。マネージド ランタイムやヘビーウェイト エンジンなしで、隅々までネイティブ コードを入手できます。
    Android アプリは標準のオープンソース ライブラリにのみ依存します。
  </FeatureCard>

  <FeatureCard title="設計によるネイティブ パフォーマンス" icon="rocket_launch" color="red" link="/docs/project-types/">
    Skip アプリは本物のネイティブ コードを使用しているため、アプリのメモリ使用量は予測可能で、
    起動時間が速く、パフォーマンス特性は
    手書きの Swift および Kotlin と同じです。
  </FeatureCard>

  <FeatureCard title="ファーストパーティ IDE" icon="code" color="purple" link="/docs/app-development/">
    Xcode をプライマリ Swift 開発環境として引き続き使用します。
    これまでどおりにビルド、デバッグ、リファクタリングします。
    プラットフォーム固有のカスタマイズには Android Studio にドロップインします。
  </FeatureCard>

  <FeatureCard title="シームレスな Android 相互運用" icon="sync_alt" color="green" link="/docs/platformcustomization/">
    特定の Android SDK またはライブラリが必要ですか？
    Skip は Kotlin および Jetpack Compose とクリーンに相互運用するため、
    フレームワークと戦うことなく Android API を統合できます。
  </FeatureCard>

  <FeatureCard title="オープンソース、長期的に構築" icon="lock_open" color="teal" link="/blog/skip-is-free/">
    Skip とそのコア フレームワークは無料でオープンソースです。
    ロックイン、独自のランタイムはなく、ベンダーが
    最新のプラットフォーム リリースに追いつくのを待つ必要もありません。
  </FeatureCard>

</div>
